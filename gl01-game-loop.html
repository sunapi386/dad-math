Game Lab · Topic 01

Make a Game from Scratch

Factorio, StarCraft, Deep Rock Galactic — under the shiny graphics they're all built on the same tiny secret. A game is a heartbeat: update the world a little, draw it, repeat — about sixty times every second. Do that fast enough and pixels come alive. Let's build one.

⏱ ~30 min   no math lesson needed first   🎮 you'll build a real game

1 · The heartbeat every game shares

Sit a person in front of a movie: it's just still pictures, flashed so fast your eye blends them. A game does the same — but instead of playing pre-made frames, it builds a fresh frame every time, using where everything is right now. That builder runs in a loop:

① read the controls → ② update the world → ③ draw everything ↺

One trip round is a tick (or a frame). Sixty ticks a second is “60 FPS.” In StarCraft each tick moves every unit a hair and checks who got shot. In Factorio each tick steps every belt and inserter. Same heartbeat, different world. Everything else is decoration.

2 · How things move: position + velocity

Here's the whole trick to movement. Give a thing a position (where it is) and a velocity (how far it jumps each tick). Then every tick, do one line:

$$\texttt{position} \;\mathrel{+}=\; \texttt{velocity}$$

That's it. Big velocity = fast. Negative velocity = leftward. Change velocity a little each tick and you get gravity, curves, bounces — everything.

🟢 The motion sandbox

Press Start the heartbeat, then drag the sliders. You're setting the ball's velocity live. It bounces off the walls by flipping that velocity — the same 3 lines of code a bouncing grenade uses.

Try: set both speeds to 0 — the ball freezes (the loop still runs, the world just isn't changing). Set sideways to 6 — it blurs across. That “blur” is your eye seeing the same ball drawn in 60 slightly-different spots each second.

3 · “Did they touch?” — collision

A game world is dumb: two things can sit in the exact same spot and nothing happens unless the code checks. For boxes the check is four comparisons — do they overlap left-right and up-down? If yes, they're touching. That one check is a bullet hitting a Zergling, a dwarf walking into lava, an item landing in a chest.

4 · 🎮 Build it: Cave Dodge

Now we glue it together — loop + velocity + collision + a state machine (the game is either ready, playing, or game over; each state behaves differently). You're a dwarf; rocks fall from the ceiling; don't get squished. Survive to score. It gets faster.

📓 Homework

  1. Name the loop in a game you love. In Deep Rock, what happens every single tick? (Hint: enemies move, your position updates, the game checks if a bug reached you.)
  2. Design a velocity. If a rock should fall twice as fast, what do you change — its position or its velocity? Why?
  3. Invent one rule. Add a rule to Cave Dodge in words: e.g. “every 20 rocks, they speed up,” or “a gold rock is worth 5.” Real game design is just piling up little rules like that.
