Secret Codes

Two thousand years ago, Julius Caesar sent battle orders his enemies couldn't read — using a trick you can learn in one minute. Today you'll encrypt like Caesar… and then break his cipher, which is the more fun half of cryptography.

⏱ ~30 min   pairs with CS 1

1 · Caesar's trick: slide the alphabet

Pick a secret number — say 3. To encrypt, slide every letter 3 steps forward: A→D, B→E, C→F… wrapping around at the end (X→A, Y→B, Z→C). The secret number is called the key. To decrypt, the receiver slides back by the same key.

$$\texttt{ATTACK AT DAWN} \;\xrightarrow{\ +3\ }\; \texttt{DWWDFN DW GDZQ}$$

2 · Your own cipher machine

🔐 Encrypt / decrypt

Type a message, slide the key. Send someone the gibberish and the key — separately!

3 · Now break it 😈

Here's the flaw. There are only 25 possible keys (sliding by 0 changes nothing). A code-breaker doesn't need to be clever — they just try all 25 and look for the one that makes words. Trying everything is called brute force.

😈 The code breaker

A secret message, encrypted with a key I'm not telling you. Scan the 25 candidates — click the row that's real English.

The big lesson: a code is only as strong as the number of keys someone must try. 25 keys = broken in a minute. Your phone's encryption has more keys than there are atoms in the universe (a number with 77 digits) — brute force would outlast the sun. Same idea as CS 1: every extra bit doubles the keys.

📓 Homework

  1. Send a real secret: encrypt a message for a family member with key 5 on paper. Give them the method but not the key — can they brute-force it?
  2. Why 25 keys and not 26? Explain in one sentence.
  3. Challenge: a spy encrypts twice — first with key 3, then encrypts the result with key 4. Is that harder to break than one key? (What single key does the same job?)
