The Sketch Studio

Everything from the last four lessons, running for real. This page carries an actual open-source rendering engine — Krbn, by Valeriu Paloș — that does live what you learned in theory: it flattens 3D onto glass, solves equations for the exact outlines, bites out or ghosts the hidden pieces, and hatches and wobbles on purpose. Your knobs. Its math. Go.

Plotter food

The download button gives you the drawing as an SVG — not a grid of pixels but a list of exact strokes. That's a file a pen plotter (a robot arm holding a real pen) can draw on real paper with real ink. It's also why you can zoom the sketch forever and it never goes blocky. Keep your favourites — if we ever get hands on a plotter, your renders are ready for it.

Genesis imperfecta

Krbn is open source (MIT licensed) and was built by Valeriu Paloș — github.com/vpalos/krbn. It was his childhood dream project, and the reason it exists is the same question this whole Lab has been circling:

“A human being can convey far more meaning in a drawing than any machine-like photorealistic render — precisely by being willing to go the other way: to renounce detail, to drop precision, even to deliberately introduce impurities and imperfections. Ah — but which detail to drop? Which imperfections to introduce? That is the fascinating part.”

You now know his engine's current answers: drop what's hidden (SK·3), keep outlines exact but optional details quiet, add wobble that's secretly deterministic (SK·4). Maybe someday you'll have better answers — the code is right there, free, for anyone to improve.

Try this tonight

  • Art show: render the same scene three ways (wobble 0 / 40 / 90), print or screenshot them, and ask the family which drawing has the most life. Tally the votes.
  • Spot the lessons: in one render, point at: a silhouette the engine solved, a stroke that got bitten, a ghost line, hatching that follows the grain. One example each.
  • Dream a scene: the engine also knows cones, helixes and imported 3D files. Sketch (with a pencil!) the scene you'd build if you wrote code for it. That drawing is a plan.
