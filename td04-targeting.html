Tower Lab · Investigation 04 · under the hood of Core Keep

Who to Shoot?

Ten bugs in range, one barrel. Every tower defense game ever made offers the same four choices — First, Last, Strongest, Closest — and most players never touch them. Tonight you'll see that each one is a single line of code, then race all four on the actual Core Keep engine, right here on this page — no graphics, thousands of steps, instant verdicts. Your gut has opinions. The lab has numbers.

⏱ ~30 min   algorithms · argmax   🎯 out-think your gut

1 · A strategy is one comparison

“Strategy” sounds like wisdom, but inside the machine each mode is just: walk the list of bugs in range, keep whichever wins one comparison. Mathematicians call it argmax — “the argument that maximizes.” Here's the real engine code you played against:

if (mode === "first"  && e.s > best.s)  best = e;  // farthest along the road
else if (mode === "last"   && e.s < best.s)  best = e;  // most recently spawned
else if (mode === "strong" && e.hp > best.hp) best = e;  // biggest health bar
else if (mode === "close"  && e.d2 < best.d2) best = e;  // smallest d² (TD·1!)

Four personalities from four comparisons: First is the firefighter (stop whoever's nearest the exit), Last is the farmer (start damage early, harvest later), Strong is the duelist (focus the scariest), Close is the sniper (shortest shot — and yes, that's the squared distance from TD·1 again, still root-free).

2 · The strategy lab

Which personality wins? Don't argue — measure. The buttons below run the real engine headless: same towers, same wave, only the mode changes. And because Core Keep has zero luck in it, these numbers are exact — run it a thousand times, get the same answer a thousand times. That's what “deterministic” buys you: experiments you can trust.

If your lab matches ours: First crushes Scenario 1 (a leak of 1 vs Last's ELEVEN — the firefighter saves the day), but in Scenario 2 Close edges First out. Why would “shortest shot” ever win? A hint: the Mortar's shell splashes, and the road passes closest to a turret exactly where bugs bunch up in its ring the longest. The deep lesson isn't “Close is good” — it's that adding one tower changed which rule was best. In systems with moving parts, the answer to “what's the best strategy?” is usually “best at what, alongside what?” — so measure, don't assume.

Overkill corner: notice Strong losing badly in both scenarios? Every Gunner piles onto the fattest bug — 5 turrets × 3 damage arrive at once, and a bug with 2 HP left still soaks all 15 (overkill, ×5). Meanwhile the skinny bugs stroll home. Rules that sound heroic can waste the most.
